### Current Development Focus

Ship and harden the **equity market map and portfolio analytics** platform: single-universe screener with a top-bar **Manage Tickers** modal (CSV Import / Paste Tickers / Current) for ingestion; Yahoo HTTP price/benchmark ingest to Postgres; market-map matrix API + heatmap UI rendered as an **expandable Sector → Sub-Theme → Ticker tree** (no row-level filter; per-row chevrons + Expand all / Collapse all toggle); portfolio weights + benchmark analytics; keep financial math in `src/domain` and services out of presentational components.

### Key Decisions / Constraints

- **Single universe:** App operates as a single "universe of stocks" screener. `GET /api/universe/default` returns or lazily creates the one active universe (DB schema remains multi-universe-capable; UI only exposes one).
- **Hierarchy (only):** Sector → Sub-Theme → Company. Drives market map rows, drill-down, rankings, and aggregations (parent metric = **average of leaf company metrics** unless later specified otherwise).
- **Trading-day semantics:** 1/5/21/63/126/252 trading days for 1D, 5D, 1M, 3M, 6M, 1Y; not calendar days.
- **Returns:** From **adjusted** close; daily return = \(P_t/P_{t-1} - 1\). Volatility: annualized **realized** vol from daily returns, \(\sigma_{ann} = \text{stdev} \times \sqrt{252}\). Sharpe uses configurable risk-free; zero-vol edge case must not divide by zero.
- **Stack:** Next.js + TypeScript, Prisma + PostgreSQL, market data **behind interfaces**; default adapter uses Yahoo **public** chart + quote HTTP (no API key; unofficial—see `docs/ARCHITECTURE.md`). Benchmarks: S&P 500, NASDAQ, DOW stored in `BenchmarkPriceHistory`.
- **Quality:** No analytics logic in React components—call services/API from UI.

### Major Changes Log

- **2026-04-18 (eve, +4):** Tightened the per-level visual rhythm on the Market Map so sectors, sub-themes, and tickers can be told apart at a glance. Each level now has its own **density** (sector rows breathe at ~0.7rem padding, sub-themes tighten to ~0.42rem, tickers compact to ~0.32rem), its own **typography** (sector = uppercase amber `#f0b65d` 700-weight with wider letter-spacing; sub-theme = cool light `#d8e3f0` 500-weight; ticker = monospace `#9eb0c8` 500-weight with letter-spacing), and its own **background tone** (`#1c2638` / `#0e1624` / `#091018`). Heat-cell font size + weight follow the same 3-step scale (0.86 → 0.84 → 0.8 rem) so each row visibly belongs to a level. The inter-sector gap row was deepened to `#06090f` and bumped to 12px so each sector card sits in a clearly visible groove. Spines + chevrons re-tuned to match (sector chevron amber, sub-theme chevron cool slate).
- **2026-04-18 (eve, +3):** Market Map redesigned around a **block-grouped hierarchy** that visually mirrors the way the user thinks about the data: each sector renders as a prominent uppercase amber header row with its own elevated background; expanding the sector reveals its sub-themes as a contiguous, slightly-recessed block sitting directly underneath, joined to the parent via a continuous "spine" (`box-shadow: inset 4px 0 0 …`) running down the sector column. Expanding a sub-theme nests ticker rows the same way, with a secondary spine through the sub-theme column. Internal seams within a block are suppressed (`borderBottom: transparent`) so each sector reads as one connected card, and a small `#0b1018` spacer row is rendered between adjacent sectors so the blocks don't visually bleed together. Sector / sub-theme / ticker rows now use distinct background tones (`#1a2333` / `#10182a` / `#0d1422`) instead of relying on row dividers. The single global control was renamed to **Show sub-themes / Hide sub-themes** (toggles only sector-level expansion, leaving ticker drill-down on the per-sub-theme chevron) so the workflow matches "compare sectors → break out into sub-themes → click a sub-theme to see tickers".

Columns reordered to **Sector | Sub-Theme | Ticker | horizons**, with sectors as the default top-level rows. Each sector row is clickable to reveal its sub-themes; each sub-theme row is clickable to reveal its tickers. A new global expand/collapse toggle was added in the controls strip. The component now performs a single `rowLevel=COMPANY` fetch and computes sector / sub-theme aggregates client-side (average of leaf metrics per horizon, matching the server logic), and applies **per-level heatmap ranges** so sector / sub-theme aggregates retain visible contrast despite their tighter spread. URL-based drill (`?sector=…&subTheme=…&rowLevel=…`) is no longer used by the UI.
